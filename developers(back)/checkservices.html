Checking Service End Points¶

When the Cymphony server is running (see developers(back)-runcommand or Run from IDE (IntelliJ)), you can test services by sending them http calls using Postman.

  • Download and install Postman.

  • Download these project configuration files intended for postman.

environment file collection file

  • Open Postman on your computer, and click Import:

  • Import both configuration files.

Cymphony Services¶

Cymphony has 11 services: cymphony-bundle-job-constraints, cymphony-bundle-job-monitor, cymphony-bundle-job-project, cymphony-bundle-job-resources, cymphony-bundle-session, cymphony-bundle-user, cymphony-bundle-util-email, cymphony-bundle-util-file, cymphony-bundle-util-invitation, and cymphony-bundle-util-log.

Below we show that how most services can be tested.

Service: Create User¶

Under cymphony/Service/User/Create User, click Send button:

Now, you can see the info in the user table, under mysql test database

Service: Algorithm Get¶

This is to get the seed for the encrypted password.

Under cymphony/Service/Session/Algortim Get, click Send button; while, there is no database operation for this.

Service: login¶

This is for a requester to sign in.

Under cymphony/Service/Session/login, click Send button:

Service: create project¶

This is for a requester to create a project. The two files, JobExtractInput.csv and Workers.csv, are in folder cymphony/back-end/resources.

Under cymphony/Service/Util/File/upload file, click Send button:

Now, you can see the info in the file_uploaded and file_uploaded_data tables.

Service: Update workflow¶

This is for a requester to upload a cymphony workflow.

Under cymphony/Service/Project/Upload workflow, click Send button:

This workflow is now in the project table, the workflow column

Service: Project Execute¶

Under cymphony/Service/Project/Project Execute, click Send button:

For each job node in the workflow, Cymphony will create a database with many tables, such as the job_cfxxkgs on the left top of the figure below. The schema of these tables in this database are copied from the public database; that is, public is a template of table schemas and all job databases will copy from.

Service: Register Worker¶

Under cymphony/Service/Job/Project/ProfileWorker/Register Worker, click Send button:

It will insert data into the profile_worker table.
