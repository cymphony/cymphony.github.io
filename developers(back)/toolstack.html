Tool Stack¶

Java¶

Cymphony uses some recent Java features, especially CompletableFuture for asynchronous computation. If you need to modify the code relevant to it, please read the guide to CompletableFuture.

Lagom¶

If you want to implement a new service in Cymphony, you need to read the following documents, and then follows the example at Example: Creating a Lagom Service.

  • Introduction to Lagom concepts

  • Lagom build philosophy

  • Splitting a system into multiple builds

  • Registering and discovering services

  • Defining a Lagom build

Other Tools¶

  • JPA: Hibernate provides a framework for mapping an object-oriented domain model to a relational database. In Cymphony, it is configured for MySQL database

  • JSON parser: JsonP : Java does not have a native JSON parser. We use Json-P for high efficiency.

  • Message-Queue: Kafka : In most of the time, Lagom will hide Kafka for workers. But, in some cases, knowing how to operate on Kafka queues is helpful.

  • Cache: Redis

  • Orchestrator: K8S

  • Project management: Maven
